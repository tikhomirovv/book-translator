# Обзор проекта

## Что это

CLI-инструмент для перевода книг и длинных текстов с помощью LLM. На вход — файл (для MVP: PDF), на выход — переведённый текст в Markdown с метаданными во frontmatter.

Проект задуман с расширяемой архитектурой: позже — веб-интерфейс, другие форматы книг, SaaS-модель — без переписывания ядра.

## Для кого

1. **Сейчас (MVP):** вы сами — проверить гипотезу, что LLM даёт достаточно качественный перевод целой книги.
2. **Далее:** заказные переводы, если качество устраивает.
3. **Потом:** SaaS для массового перевода книг.

## Проблема

- Книга не помещается в контекстное окно модели — нужен пошаговый перевод с сохранением смысла.
- Из PDF и других форматов текст нужно извлечь без потери структуры.
- При длинном переводе теряется контекст (имена, термины, стиль) — нужна «память» по ходу работы.
- Перевод может прерваться — нужен возобновляемый прогресс.

## Ценность

- Автоматизация перевода длинных документов без ручной разбивки на куски.
- Умное накопление контекста (глоссарий, суммаризация, overlap) для более согласованного перевода.
- Возможность продолжить с места остановки по идентификатору перевода.
- Гибкие промпты и настройки модели без пересборки программы.

## Текущий статус

**Фаза:** MVP — CLI работает, идёт доводка качества перевода и извлечения текста.

**Зафиксировано:**
- Стек: **Go**, CLI (Cobra), конфигурация **Viper**.
- MVP: PDF → перевод → Markdown; CLI + команда `extract` для отладки извлечения.
- Главная сущность — **перевод** (Translation, UUID v4), а не книга.
- Архитектура: Standard Go Layout + Clean Architecture (ports/adapters) — см. `.docs/technical-design.md`.
- PDF: `razvandimescu/gopdf` с reflow параграфов; токены/стоимость — из ответа API, tiktoken позже.
- Чанк по умолчанию: **10 параграфов**.
- LLM: два независимых профиля в конфиге — `llm.translation` и `llm.context`.
